## Project Goal

Create a local-first cohort variant browser for WGS and WES data. The system should ingest VCF/GVCF
files, store genotypes in Hail VDS, compute database-level allele frequencies, and expose variant
and gene-level frequency queries through a patched gnomAD browser.

## Stable Decisions

- VDS is the durable genotype store.
- Annotated MatrixTable is the downstream annotation and export layer.
- Cohort AF comes from Hail `variant_qc()` and should expose `ac_total`, `an_total`, `af_total`,
  and `hom_count`.
- Browser serving uses Elasticsearch index `cohort_variants`.
- The tracked source for browser changes is `browser/`; `gnomad-browser/` is generated by
  `setup.sh`.
- Current reference assumptions from project docs: GRCh37/hg19, VEP 108, gnomAD v2.1.1 exomes.

## Metadata Rules

- `sample_id` is the join key between genomic data, sample metadata, and panels.
- Panels are represented as one row per sample and panel. Multiple rows for one sample are valid.
- Preserve both social self-defined sex and chromosomal sex as separate concepts.
- Validate panels-only sample IDs and likely sample ID typos before merging metadata.
- Normalize date fields before loading them into Hail or Elasticsearch.

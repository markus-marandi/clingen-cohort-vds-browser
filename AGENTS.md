# AGENTS.md

## Purpose

Build a local-first WGS/WES cohort browser that ingests VCF/GVCF files, stores genotypes in
Hail VDS, computes cohort variant frequencies, and exposes variant and gene-level queries through
a patched public gnomAD browser.

## Architecture

```text
raw GVCFs -> bcftools filter -> Hail VDS -> annotated MatrixTable -> Elasticsearch -> gnomAD browser
```

- `parallel_ingest_cohort.py` preprocesses GVCFs and incrementally combines them into VDS.
- `annotate_cohort.py` converts VDS to MatrixTable, runs annotation, and calculates cohort AF with
`hl.variant_qc()`.
- `browser/data-pipeline/cohort_export.py` exports variant documents to Elasticsearch.
- `browser/` contains tracked patches for upstream `gnomad-browser/`.
- `gnomad-browser/` is generated by `setup.sh` and is not source of truth.

## Durable Context

Read `MEMORY.md` first, then only the scoped memory file for the area you touch:

- `docs/pipeline/MEMORY.md` for Hail, VDS, MatrixTable, metadata, and AF rules.
- `browser/MEMORY.md` for Elasticsearch, GraphQL, and gnomAD browser assumptions.

Use `TODO.md` as the roadmap index. Use scoped `TODO.md` files for implementation details.

## Local Setup

Run from the repository root:

```bash
./setup.sh
```

This clones upstream gnomAD browser into `gnomad-browser/`, overlays files from `browser/`, and
installs browser dependencies.

## Pipeline Commands

Ingest:

```bash
python parallel_ingest_cohort.py \
    --raw-gvcf-dir /path/to/raw_gvcfs \
    --filtered-gvcf-dir /path/to/filtered_gvcfs \
    --output-vds-dir /path/to/vds_outputs \
    --temp-base /path/to/tmp/combiner_temp \
    --manifest-path /path/to/ingest_manifest.json \
    --n-cores 16 \
    --memory-gb 64
```

Annotate and compute database-level AF:

```bash
python annotate_cohort.py \
    --vds-path /path/to/cohort.vds \
    --output-mt /path/to/cohort_annotated.mt \
    --metadata-path /path/to/metadata.csv \
    --n-cores 16 \
    --memory-gb 64 \
    --overwrite
```

Export:

```bash
python browser/data-pipeline/cohort_export.py \
    --mt-path /path/to/cohort_annotated.mt \
    --es-url http://localhost:9200 \
    --index cohort_variants
```

Browser:

```bash
cd gnomad-browser
docker compose up --build
```

Expected local services:

- Elasticsearch: `http://localhost:9200`
- GraphQL API: `http://localhost:8000`
- Browser UI: `http://localhost:3000`

## Testing Expectations

There is no packaged automated test suite yet. For now, verify changes with the smallest relevant
smoke check:

- Python syntax/import check for edited Python files.
- Hail fixture or tiny local demo when touching ingest, annotation, export, or metadata behavior.
- Elasticsearch index/query check when touching `browser/data-pipeline/`.
- Docker Compose browser smoke test when touching `browser/graphql-api/` or Compose files.

Document any skipped check in the final response.

## Data Rules

- Keep social self-defined sex from metadata separate from chromosomal sex inferred from genotypes.
- Join clinical metadata by `sample_id`.
- Treat panels as one row per `sample_id` and `panel`; multiple rows per sample are expected.
- Keep raw genotype storage in VDS; write annotations and query/export fields to MatrixTable.
- Calculate cohort-level `ac_total`, `an_total`, `af_total`, and `hom_count` from Hail
`variant_qc()`.

## Coding Guidance

- Prefer small, direct changes that match existing scripts and docs.
- Do not commit raw VCF/GVCF, VDS, MatrixTable, Hail logs, or generated `gnomad-browser/` output.
- Do not add credentials or internal server details to tracked docs.
- Keep root docs concise; move area-specific details into scoped memory and TODO files.
